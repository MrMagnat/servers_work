# AI News Monitoring Agent

Автономный агент мониторинга новостей о реальных внедрениях ИИ в бизнес. Запускается 4 раза в день, фильтрует дубли, оценивает релевантность через LLM и отправляет дайджест в Telegram с кнопками обратной связи.

## Возможности

- Поиск через Exa.ai (нейросетевой) + Google News RSS как резерв
- LLM-оценка релевантности (Gemini Flash) — только реальные кейсы с результатами
- Семантическая дедупликация на основе эмбеддингов
- Telegram-бот с кнопками: отклонить, сгенерировать пост, написать статью-кейс
- Память отклонённых паттернов: агент учится фильтровать то, что не интересует
- Генерация постов (Claude Haiku) и статей-кейсов (Claude Sonnet)

## Установка

### 1. Клонирование и зависимости

```bash
cd ai-news-agent
pip install -r requirements.txt
```

### 2. Настройка окружения

```bash
cp .env.example .env
```

Заполни `.env`:

| Переменная | Где взять |
|---|---|
| `OPENROUTER_API_KEY` | [openrouter.ai/keys](https://openrouter.ai/keys) |
| `EXA_API_KEY` | [exa.ai/dashboard](https://exa.ai/dashboard) |
| `TELEGRAM_BOT_TOKEN` | [@BotFather](https://t.me/BotFather) → `/newbot` |
| `TELEGRAM_CHAT_ID` | [@userinfobot](https://t.me/userinfobot) — напиши ему, получишь свой ID |
| `TAVILY_API_KEY` | [tavily.com](https://tavily.com) (опционально) |

### 3. Запуск

**Обычный режим** (планировщик + Telegram-бот):
```bash
python main.py
```

**Тест-режим** (один цикл без Telegram, вывод в консоль):
```bash
python main.py --test
```

## Расписание

Агент запускается автоматически в **08:00, 12:00, 16:00, 20:00 по московскому времени**.

При первом запуске цикл выполняется немедленно.

## Структура проекта

```
ai-news-agent/
├── .env                    # секреты (не коммитить!)
├── .env.example            # шаблон переменных
├── requirements.txt

├── scheduler.py            # cron-логика, run_agent_cycle()

├── processor.py            # дедупликация, LLM-фильтр, сортировка
├── memory.py               # SQLite: отправленные, отклонения, лог
├── llm.py                  # OpenRouter: scoring, embedding, генерация
├── bot.py                  # Telegram-бот, inline-кнопки
├── templates.py            # шаблоны и промпты
├── config.py               # загрузка .env
├── agent.log               # лог с ротацией 7 дней (создаётся при запуске)
└── database/
    └── agent.db            # SQLite (создаётся автоматически)
```

## Обратная связь через Telegram

Каждая новость отправляется с тремя кнопками:

- **❌ Не актуально** — бот попросит комментарий, извлечёт паттерн и запомнит его. При следующих запусках похожие новости будут отфильтрованы.
- **📢 Пост в ТГ** — генерирует готовый пост для Telegram-канала (Claude Haiku).
- **📄 Статья/Кейс** — пишет структурированную статью-кейс для публикации (Claude Sonnet).

## Логи

Логи пишутся в `agent.log` с ротацией раз в сутки, хранятся 7 дней. Дублируются в `stdout`.
